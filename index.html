Why procedural memory?

The missing layer between
raw prompts and fine-tuning.

Small language models are capable but forgetful. Every interaction is stateless — the model sees each prompt as if for the first time. coding-intern adds a persistent learning layer that captures, organizes, and reuses knowledge across sessions.

⊞

Prompts are not memory

Without a memory layer, every task starts from scratch. Manual prompt engineering is brittle — skills are trapped in text, not structured for reuse. coding-intern treats each interaction as a structured experience with learning signals.

⊡

Fine-tuning is expensive

Full fine-tuning costs hundreds in compute and requires curated datasets. coding-intern learns online — from real usage — and consolidates into lightweight skills without gradient updates.

≋

RAG has no learning curve

Retrieval-Augmented Generation fetches static documents. It doesn't improve with use. coding-intern's skill graph gets denser and smarter over time as experiences cluster into higher-level abilities.

⟐

Local-first by design

No cloud dependency, no data leaving your machine. Runs on CPU, Apple Silicon, and NVIDIA GPUs. The skill store is a local SQLite database — portable, inspectable, and yours.

how it works

Capture. Cluster. Consolidate. Improve.

01

Every interaction is an experience

Each task → execution → result is recorded with learning signals: novelty, outcome value, and frequency. Stored in a local SQLite buffer.

02

Similar experiences cluster together

Embeddings are computed via sentence-transformers, then clustered by cosine similarity. Clusters of 3+ related experiences become skill candidates.

03

Clusters become reusable skills

Each cluster generates a prompt template, trigger keywords, and verification checks — compressed into a SkillNode stored in the skill database.

04

The router improves automatically

New skills are indexed in FAISS for fast retrieval. Over time, more tasks are handled locally without LLM fallback — the system gets faster and cheaper.

quick start

Up and running in minutes.

Requires Python 3.11+ and Ollama. No GPU required.

1

Install the package

Install directly from GitHub using pip.

pip install git+https://github.com/Asanali111/coding-intern.git
2

Run hardware setup

Auto-detect your hardware and configure the optimal model for your system.

neural-skill hardware setup
3

Start learning

Run your first task — the system captures it as an experience and routes it through available skills.

neural-skill run "summarize the key points from this article"
4

Watch it improve

Check stats and watch the skill graph grow as the system consolidates experiences into reusable skills.

neural-skill stats && neural-skill graph

Works on macOS, Linux, and Windows. See the GitHub repo for detailed setup instructions.

faq

Common questions.

What makes this different from RAG or fine-tuning?

+

RAG retrieves static documents — it never improves with use. Fine-tuning requires curated datasets and expensive compute. coding-intern learns incrementally from real interactions, clustering experiences into reusable skills without any gradient updates. It sits in between — lightweight, online, and persistent.

What hardware do I need?

+

A CPU with 4+ cores and 8GB of RAM is sufficient for basic usage. Apple Silicon (M1+) and NVIDIA GPUs are supported for accelerated embedding computation. The system automatically detects your hardware and recommends the optimal configuration.

Does it require an internet connection?

+

No. Everything runs locally — experience storage (SQLite), embedding computation (sentence-transformers), and skill routing. The only optional network dependency is the LLM fallback if you choose a cloud-hosted model over a local one like Ollama.

How are skills stored? Can I inspect them?

+

Skills are stored as structured JSON records in a local SQLite database at ~/.neural-skill/. Each skill contains a prompt template, trigger keywords, verification checks, and metadata. You can inspect, edit, or delete them directly.

Can I use it with any language model?

+

Yes. coding-intern uses litellm as its LLM abstraction layer, which supports 100+ models — Ollama, OpenAI, Anthropic, Together AI, and more. You can configure the backend in a single config file. The default setup uses local Ollama models.

How do I contribute or report issues?

+

The project is open source under the MIT license. You can open issues, submit pull requests, or start discussions on the GitHub repository. Contributions of all kinds — code, docs, bug reports — are welcome.
